    Introduction

    Welcome to the PostIT API! You can use our API to access PostIT API endpoints, which allows users to sign up , sign in , create a group , post message to a group and so on.

    Development

    The application was developed with NodeJs and Express is used for routing. The Postgres database was used with sequelize as the ORM.

    Installation

    First ensure you have - node installed or install Node - postgres installed on your local machine you find out more here then Clone the project repository from your terminal git clone https://github.com/mikey2020/post-it Change directory into the post-it directory Run npm install to install the dependencies in the package.json file Run npm run dev:server to start the project

    Users

    List Users

    To get list of all registered users

    Request

    • Endpoint: POST: api/v1/users
    • Requires: Authentication

    Response

    • Status: 200: OK
    • Body (application/json)
      List all users
   {
     "users": [{
       "id": 1,
       "username": "sasuke",
       "email": "sasuke@konoha.com"
       "phoneNumber": "09012345678"
     },
     {
       "id": 2,
       "username": "naruto",
       "email": "naruto@konoha.com"
       "phoneNumber": "03032345676"
     }],
     "message": "naruto signed in"
   }

    Request

    • Endpoint: POST: /api/v1/user/signup
    • Body (application/json)

    Response

    • Status: 200: OK
    • Body (application/json)

{

    "email": "boruto@uzumaki.com",

    "password": "sarada",
    "passwordConfirmation": "sarada"

   signupResponse: { 
       "message": "boruto successfully added", 
       "userToken": "token"
    }
}

    Status codes

    This sections is to let you know the different kinds of status codes used in this API

    The PostIt API uses the following status codes:

    Status Code Meaning
    200 OK - Your request was successful.
    201 Created - Your request was successful and has resulted in a new resource being created.
    400 Bad Request -- Your request is not correct.
    401 Unauthorized -- You have not entered correct signin/login details.
    403 Forbidden -- You trying to access a resource that is restricted.
    404 Not Found -- Your request could not be found on the server.
    500 Internal Server Error -- We had a problem with our server. Try again later.
